# Создание статического шаблона для печати карт полковников

## Почему
Необходимо создать статичный HTML+CSS шаблон для печати уникальных карточек юнитов (полковников) на листах формата A4. Шаблон должен обеспечивать идеальное совпадение лицевой стороны и рубашки при двусторонней печати без использования JS и масштабирования.

## Что изменится
- Создание новой папки `UniqueColonel` для хранения шаблона
- Создание HTML-файла `index.html` со статической разметкой 11 карточек на 4 страницах A4
- Создание CSS-файла `style.css` с настройками `print-ready` для точного позиционирования в миллиметрах
- Размещение сетки 3x3 (9 карт на страницу) строго по центру листа
- Добавление меток реза (crop marks) только на лицевую сторону
- Настройка bleed (вылет под обрез) +2mm по внешнему краю всей сетки
- Настройка двусторонней печати: чередование страниц "Лицевая сторона" -> "Рубашка"

## Влияние
- Affected specs: Печать карточек полковников
- Affected code: Новые файлы в `UniqueColonel/`

## ADDED Requirements
### Requirement: Статичный Print-Ready Шаблон
Система ДОЛЖНА предоставлять HTML-файл с CSS, оптимизированный для печати.

#### Scenario: Подготовка к печати
- **WHEN** пользователь открывает HTML-файл в браузере и вызывает печать
- **THEN** формируются 4 страницы A4 (лицо, рубашка, лицо, рубашка) без отступов браузера и масштабирования.

### Requirement: Точная Сетка и Размеры
Система ДОЛЖНА использовать строго заданные размеры: карта 63.5x88.9 мм, отступы между картами 0 мм.

#### Scenario: Двусторонняя печать
- **WHEN** пользователь печатает документ на обеих сторонах листа
- **THEN** рубашки идеально совпадают с лицевой стороной благодаря точной центровке сетки на листе A4.

### Requirement: Метки реза и Bleed
Система ДОЛЖНА отображать метки реза (#cccccc, 0.2mm) на лицевых страницах и обеспечивать буфер (bleed) вокруг сетки.

#### Scenario: Обрезка карт
- **WHEN** пользователь режет распечатанный лист по меткам
- **THEN** карты имеют точный размер 63.5x88.9 мм без белых полей по краям.
