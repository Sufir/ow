# Code Wiki: Faction Card Generator

## 1. Назначение проекта
Проект представляет собой браузерный инструмент для создания, редактирования и подготовки к печати (в формате A4) игровых компонентов для настольной игры (предположительно в сеттинге BattleRobots / MoonSystems).
Позволяет генерировать:
- Планшеты фракций (Faction Cards) с описанием юнитов, свойств, начальных условий и иллюстрациями (лицевая и задняя стороны).
- Мини-карты (Mini American Cards, 41×63 мм) с различными рубашками, текстом и иконками.

Все данные сохраняются локально в браузере (`localStorage`) с возможностью экспорта и импорта в виде JSON-файлов.

## 2. Общая архитектура проекта
Проект реализован полностью на стороне клиента (Client-Side) без использования сборщиков (Webpack/Vite) или JS-фреймворков (React/Vue). 
Используется Vanilla JavaScript (ES6+), HTML5 и CSS3.

Архитектурно проект разделен на два независимых модуля (приложения), каждое из которых имеет свои HTML-шаблоны и скрипты:
1. **Модуль Faction Card** (`Template.html`, директория `faction-card/`) — редактор двусторонних планшетов фракций.
2. **Модуль Mini Cards** (`MiniAmericanCards.html`, `MiniAmericanCardsLandscape.html`, директория `mini-cards/`) — редактор сетки мини-карт для печати.

Хранение состояния реализовано через паттерн "Автосохранение" (Debounce) в `localStorage`. Взаимодействие с пользователем построено на нативных API: `contenteditable` для текста, Drag & Drop и `FileReader` для работы с локальными изображениями.

## 3. Обязанности основных модулей

### 3.1 Модуль "Планшет фракции" (`faction-card`)
- **`Template.html`**: Главная разметка планшета (лицевая и обратная сторона). Включает тулбар, таблицу юнитов, боковые панели для свойств и подготовки, а также SVG-оверлей для визуализации связей.
- **`app.js`**: Контроллер пользовательского интерфейса планшета. Обрабатывает события кнопок тулбара, управляет строками таблицы юнитов (добавление, удаление, группировка, разделители) и панелью "Подготовка" (роботы, нефть, очередь).
- **`storage.js`**: Слой работы с данными (`StorageAPI`). Отвечает за сохранение и восстановление всех аспектов планшета (таблицы, свойств, связей, информации о фракции и заднике) в `localStorage`, а также за формирование и парсинг JSON-файлов экспорта/импорта.
- **`props.js`**: Управляет боковой панелью "Свойства юнитов". Реализует добавление, удаление, редактирование и свободное перемещение (Drag & Drop) плавающих блоков со свойствами.
- **`links.js`**: Отвечает за визуальное соединение блоков свойств с конкретными юнитами в таблице с помощью SVG-линий (`<polyline>`). Рассчитывает координаты и перерисовывает линии при скролле или изменении размера.
- **`style.css`**: Стилизация планшета фракции, включая медиа-запросы для точной печати (`@media print`).

### 3.2 Модуль "Мини-карты" (`mini-cards`)
- **`MiniAmericanCards.html` / `MiniAmericanCardsLandscape.html`**: HTML-шаблоны (книжная/альбомная ориентация) для сетки карт.
- **`cards.js`**: Логика генерации парных листов (лицо/рубашка) карт с номерами. Рассчитывает позиции меток реза (cut marks), управляет переворотом рубашки (зеркалирование) и экспортом/импортом своего состояния.
- **`cards-landscape-sixes.js`**: Расширенная логика для альбомной ориентации (добавление по 6 карт). Добавляет управление размером текста, шрифтами, фоновыми изображениями и типами карт (Goal / Tech).
- **`cards.css` / `cards-landscape-sixes.css`**: Стили, задающие точные физические размеры в миллиметрах (`mm`) для корректной печати компонентов на A4.

### 3.3 Статические ресурсы
- **`Fonts/`**: Пользовательские шрифты (AlienEncounters, Oil Wars, OpenGostTypeB и др.), подключаемые через `@font-face`.
- **`Units/`, `BattleRobots/`, `Goals-Tachs/`**: Наборы дефолтных изображений и иконок (юниты, роботы, фоны).
- **`Saved/`**: Директория с примерами сохраненных JSON-файлов для различных фракций игры (например, "Братство Сингулярности.json", "Клонэйд Ресёрч.json").

## 4. Описания ключевых классов и функций

### `faction-card/storage.js`
Глобальный объект `StorageAPI` инкапсулирует логику работы с хранилищем:
- `init(config)`: Инициализирует префиксы ключей localStorage для изоляции данных разных документов.
- `makeSaveTableDebounced(tbody)`: Возвращает функцию для отложенного (debounce 150ms) сохранения состояния DOM-таблицы в JSON.
- `exportTable()`: Собирает данные из всех панелей (таблица, свойства, связи, задник) в единый JSON-объект и вызывает нативное скачивание файла.
- `importTableFromFile(file, ...renderFns)`: Читает JSON-файл через FileReader и вызывает переданные функции-рендеры (`renderTable`, `renderProps`, `renderLinks` и т.д.).

### `faction-card/app.js`
- `renderTable(rows)`: Полностью перестраивает DOM-дерево таблицы юнитов на основе массива данных. Восстанавливает разделители, комментарии, метки групп.
- `buildUnitCell(caption, imgSrc)`: Генерирует DOM-структуру для ячейки юнита (содержит `img` с зоной Drag & Drop и редактируемую подпись `contenteditable`).
- `initDropzone(zone)`: Навешивает обработчики событий `dragover`, `drop`, `click` и `paste` (для перехвата картинок из буфера обмена) на элементы изображений.

### `faction-card/links.js`
Глобальный объект `UnitLinks`:
- `getAnchors(propId, unitId)`: Вычисляет абсолютные координаты точек привязки между блоком свойства и соответствующей строкой юнита.
- `updateAll()`: Пересчитывает и обновляет атрибуты `points` для всех SVG-линий (`<polyline>`), обеспечивая их привязку при прокрутке панели или ресайзе окна.
- `createLink(propId, unitId)`: Создает новую SVG-линию соединения и сохраняет связь в локальном состоянии.

### `faction-card/props.js`
Глобальный объект `UnitProps`:
- `hydratePropItem(propEl)`: Инициализирует логику выделения, редактирования текста (очистка от стилей при вставке) и Drag & Drop для карточки свойства.
- `startDrag`, `onDragMove`, `endDrag`: Набор обработчиков Pointer Events для кастомного, плавного перетаскивания блоков свойств по вертикальной оси внутри панели.

## 5. Взаимосвязи зависимостей
Проект **не имеет** внешних NPM-зависимостей или библиотек (отсутствуют React, jQuery, lodash и т.д.). Зависимости внутри модулей реализованы через глобальную область видимости (Window):
- Скрипты `app.js`, `props.js`, `links.js` жестко зависят от глобального объекта `StorageAPI` (из `storage.js`) для сохранения своих данных.
- Скрипт `links.js` зависит от DOM-узлов, созданных в `app.js` (подписи юнитов) и `props.js` (блоки свойств), связывая их через уникальные атрибуты `data-unit-id` и `data-prop-id`.
- Порядок подключения скриптов в HTML-файлах (например, `Template.html`) критичен: сначала `storage.js`, затем `app.js`, `props.js`, `links.js`.

## 6. Инструкции по поддержке и развитию

### Запуск проекта для разработки
Поскольку проект состоит из чисто статических файлов, его можно открыть, просто дважды кликнув по `Template.html` или `MiniAmericanCards.html` в браузере. Для избежания потенциальных CORS-ошибок при локальной работе с модулями/шрифтами рекомендуется использовать простой локальный веб-сервер:
```bash
npx http-server .
# или
python3 -m http.server 8000
```

### Добавление новых полей в Планшет Фракции
1. Добавьте HTML-разметку элемента с атрибутом `contenteditable="true"` или `data-key="..."` в нужную панель в `Template.html`.
2. В файле `storage.js` обновите функции сериализации данных (например, `buildFactionInfoFromPanel`) и функции восстановления DOM (`restoreFactionInfoInto`), чтобы они учитывали ваше новое поле.
3. В файле `app.js` добавьте слушатель события `input` на ваше новое поле (или обновите существующий делегированный слушатель) для вызова соответствующей debounced-функции сохранения (например, `saveFactionInfo()`).

### Отладка SVG-связей
Если линии связи (в модуле `links.js`) позиционируются неверно, проверьте логику внутри функции `getAnchors()`. Она опирается на нативный метод `getBoundingClientRect()`. Убедитесь, что родительские контейнеры (особенно `pageEl` и `panelEl`) не имеют CSS-трансформаций (`transform: scale`, `translate`), которые могут исказить расчет абсолютных координат для SVG-оверлея.

### Инструкции по печати (Экспорт в PDF)
Для правильной генерации PDF или вывода на бумагу формата A4 важно сохранять CSS-правила внутри блока `@media print` в `style.css` и `cards.css`. В настройках печати браузера (Chrome / Edge) пользователю необходимо:
- Включить опцию **"Фоновая графика" (Background graphics)**.
- Установить **поля (Margins)** в значение "Нет" (None) или "Минимальные".
- **Масштаб (Scale)** установить на 100% или "По размеру страницы" (зависит от настроек драйвера принтера).
